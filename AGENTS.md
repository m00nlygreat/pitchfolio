# AGENTS

## UI Copy Rules

- Do not put PRD text, planning notes, workflow philosophy, or implementation explanations into product UI.
- Do not explain internal stages, permissions, database behavior, aggregation logic, or scoring formulas in user-facing screens.
- UI text must focus only on the user's current action, current state, and the minimum data needed to complete the task.
- Prefer short product copy such as labels, actions, and brief status messages over explanatory paragraphs.
- If a sentence reads like documentation, an admin manual, or a product spec, it does not belong in the UI.
